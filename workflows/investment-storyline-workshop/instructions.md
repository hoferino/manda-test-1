# Investment Storyline Workshop - Execution Instructions

## Overview

This workflow facilitates an **interactive storyline development session** with the user to create a compelling investment narrative for CIM creation. The workflow follows a **5-Act structure** (Discovery â†’ Synthesis â†’ Construction â†’ Validation â†’ Documentation) and emphasizes **story-first, slides-second** philosophy.

## Primary Agent

**Story Architect** leads this workflow, coordinating with:
- **Information Vault** - for data retrieval and fact verification
- **Finance Analyst** - for financial insights and value drivers
- **Company Analyst** - for business strengths and competitive positioning

## Workflow Philosophy

**Story Before Slides**: Develop the narrative in prose form before thinking about PowerPoint. A coherent story translates into great slides; great slides without a story create confusion.

**Collaborative, Not Dictated**: The Story Architect facilitates dialogue with the user. The user knows the business; the agent structures that knowledge into a compelling narrative.

**Evidence-Based**: Every claim must link to supporting data from the Information Vault. No hand-waving or unsubstantiated statements.

**Buyer-Focused**: The storyline must resonate with the target buyer type and address their key priorities.

## Execution Flow

---

### **ACT 1: DISCOVERY** (Understanding the Business)

**Objective:** Gather the user's perspective and understand what makes this company special

**Steps:**

1. **Welcome and Context Setting**
   - Greet the user and explain the workshop process
   - Confirm target buyer type (from workflow inputs)
   - Set expectations for workshop duration

2. **Open-Ended Business Discussion**
   - Ask: "In your view, what makes this company special or attractive to buyers?"
   - Ask: "What are the 2-3 biggest strengths of this business?"
   - Ask: "What's the primary growth story here?"
   - Ask: "Are there any concerns or risks we need to address in the storyline?"

3. **User Hypothesis Capture**
   - Listen for themes the user emphasizes
   - Note any specific storylines or angles the user suggests
   - Identify what the user is most excited about

4. **Initial Data Gathering Request**
   - Request comprehensive analysis from Finance Analyst:
     - "Finance Analyst: Provide summary of financial performance, key value drivers, growth trends, and valuation considerations"
   - Request business intelligence from Company Analyst:
     - "Company Analyst: Provide analysis of business strengths, competitive positioning, customer value proposition, and growth opportunities"
   - Request supporting data from Information Vault as needed

**Output from Act 1:**
- User's perspective on company strengths and storyline angles
- Comprehensive financial and business intelligence from specialist agents
- Initial understanding of what matters most

---

### **ACT 2: SYNTHESIS** (Identifying Themes)

**Objective:** Analyze all inputs and propose 3-5 core storyline themes

**Steps:**

1. **Data Analysis**
   - Review financial analysis from Finance Analyst
   - Review business insights from Company Analyst
   - Identify patterns and standout strengths
   - Note any contradictions or concerns

2. **Theme Identification**
   - Synthesize user input + agent analysis into 3-5 core themes
   - Examples of themes:
     - "Market-leading position with defensible competitive advantages"
     - "Exceptional financial performance with strong growth trajectory"
     - "Proven management team with deep industry expertise"
     - "Significant white space opportunities for expansion"
     - "Mission-critical product with high customer retention"

3. **Theme Presentation**
   - Present proposed themes to user with supporting evidence
   - Format:
     ```
     **Proposed Core Themes:**

     1. **[Theme 1]**: [Description]
        - Supporting Evidence: [Key facts with sources]
        - Why This Matters to Buyers: [Strategic relevance]

     2. **[Theme 2]**: [Description]
        - Supporting Evidence: [Key facts with sources]
        - Why This Matters to Buyers: [Strategic relevance]

     [Continue for 3-5 themes]
     ```

4. **Collaborative Refinement**
   - Ask: "Do these themes resonate with you?"
   - Ask: "Are we missing anything critical?"
   - Ask: "Should we prioritize any theme differently?"
   - Refine based on user feedback

5. **Theme Prioritization**
   - Work with user to rank themes by importance
   - Identify the "headline" theme (what buyers should remember most)

**Output from Act 2:**
- 3-5 validated core storyline themes, prioritized
- Supporting evidence for each theme with sources
- Clear understanding of what matters most to target buyers

---

### **ACT 3: CONSTRUCTION** (Building the Narrative)

**Objective:** Construct the full investment storyline with narrative arc

**Steps:**

1. **Investment Thesis Statement**
   - Draft investment thesis using Opportunity-Value-Potential-Proof-Fit framework
   - Present to user:
     ```
     **Draft Investment Thesis:**

     [Company Name] is a [market position] [business type] serving [target market].

     **The Opportunity:** [What is this company?]
     **The Value:** [Why is it valuable - performance and positioning]
     **The Potential:** [What could it become - growth opportunities]
     **The Proof:** [Why should buyers believe it - evidence]
     **Ideal Buyer Profile:** [Who should buy and why]
     ```
   - Refine based on user feedback

2. **Narrative Arc Development**
   - Structure the story flow:
     - **Opening (The Opportunity)**: How we introduce the company and grab attention
     - **Rising Action (The Business)**: What they do, how they do it, why customers buy
     - **Climax (The Value Creation)**: Performance to date and future potential
     - **Resolution (The Investment)**: Why this is attractive, who should buy, what's next

3. **Message Hierarchy**
   - Develop 4-level message structure:
     - **Level 1 (Headline)**: The one thing buyers remember
     - **Level 2 (Pillars)**: 3-5 core themes
     - **Level 3 (Evidence)**: Key facts supporting each pillar
     - **Level 4 (Details)**: Supporting context for deep divers

4. **Evidence Mapping**
   - For each claim in the storyline, map to source:
     - Request verification from Information Vault
     - "Information Vault: Verify [claim] and provide source citation"
   - Create evidence map showing claim â†’ source linkages
   - Flag any claims lacking supporting data

5. **Risk Acknowledgment**
   - Identify potential buyer concerns
   - Develop how to address in narrative (acknowledge + mitigate)
   - Examples:
     - Customer concentration â†’ "Yes, but retention is 95% and diversification underway"
     - Management depth â†’ "Experienced leadership + succession planning in place"

**Output from Act 3:**
- Complete investment thesis statement
- Structured narrative arc
- Message hierarchy (headline â†’ pillars â†’ evidence â†’ details)
- Evidence map linking claims to sources
- Risk mitigation strategy

---

### **ACT 4: VALIDATION** (Testing the Story)

**Objective:** Ensure storyline is coherent, compelling, and buyer-relevant

**Steps:**

1. **Coherence Check**
   - Review narrative flow: Does opening lead logically to resolution?
   - Check theme consistency: Do all pieces support the same story?
   - Validate evidence: Is every claim backed by data?

2. **Buyer Persona Test**
   - Review storyline through lens of target buyer type:
     - **Strategic Buyers**: Does it show synergy potential and strategic fit?
     - **Financial Buyers**: Does it show growth runway and cash flow potential?
     - **International Buyers**: Does it show market entry advantage?
     - **Competitors**: Does it show consolidation value?
   - Ask user: "If you were a [target buyer type], would this resonate?"

3. **Clarity and Simplicity Check**
   - Is the storyline clear and easy to understand?
   - Have we avoided jargon and overcomplexity?
   - Can someone explain this company in 60 seconds using our storyline?

4. **Gap Identification**
   - What claims lack strong evidence?
   - What questions might buyers ask that we haven't addressed?
   - What additional data would strengthen the storyline?

5. **User Confirmation**
   - Present complete storyline for final review
   - Ask: "Does this storyline feel right?"
   - Ask: "Is this how you want buyers to think about this company?"
   - Make final refinements

**Output from Act 4:**
- Validated storyline that passes coherence, buyer relevance, and clarity tests
- Identified gaps for future data collection
- User confirmation and buy-in

---

### **ACT 5: DOCUMENTATION** (Capturing the Storyline)

**Objective:** Generate deliverables for use in CIM creation and marketing

**Steps:**

1. **Storyline Brief Creation**
   - Generate comprehensive storyline brief (2-3 pages):
     - Investment thesis statement
     - Core themes with evidence
     - Narrative arc
     - Message hierarchy
     - Buyer positioning
     - Evidence map
     - Risks and mitigations
   - Save to output location

2. **Investment Thesis One-Pager** (Always)
   - Standalone investment thesis document
   - Use for quick reference and alignment
   - Format: Opportunity-Value-Potential-Proof-Fit

3. **Teaser Generation** (If `include_teaser = true`)
   - Create one-page investment teaser using storyline
   - Include:
     - Opportunity statement
     - Company overview
     - Investment highlights (from core themes)
     - Financial snapshot
     - Strategic value
     - Transaction overview
   - All claims sourced from evidence map

4. **CIM Narrative Outline** (If `include_cim_outline = true`)
   - Generate prose outline for full CIM:
     - Executive Summary (narrative, not bullets)
     - The Business Story (products, customers, operations)
     - The Value Creation Story (performance, opportunities)
     - The Financial Story (historicals, projections, valuation)
     - The Investment Opportunity (why buy, strategic fit, next steps)
   - This is PROSE (paragraphs), not slide bullets
   - Becomes foundation for eventual slide deck

5. **Evidence Map Export**
   - Generate evidence map document
   - Shows every claim â†’ source linkage
   - Quality control tool for fact-checking

6. **Next Steps Recommendation**
   - Based on identified gaps, recommend:
     - Additional data to collect
     - Workflows to run (financial analysis, competitive analysis, etc.)
     - CIM creation timeline
   - Suggest: "Would you like me to begin converting this narrative into a full CIM?"

**Output from Act 5:**
- Storyline brief (comprehensive)
- Investment thesis one-pager
- Investment teaser (optional)
- CIM narrative outline (optional)
- Evidence map
- Next steps recommendation

## Special Handling

### If Data is Limited

If the data room has insufficient data for robust storyline:
1. Use Act 1 Discovery to gather user knowledge
2. Flag data gaps clearly in Act 2 Synthesis
3. Build storyline with available data + user input
4. Mark claims as "User-Provided (Pending Verification)" in evidence map
5. Recommend running data room audit workflow
6. Suggest specific documents to request from company

### If User Disagrees with Themes

If user pushes back on proposed themes:
- Ask probing questions: "What would you emphasize instead?"
- Request user to propose alternative themes
- Validate user themes against available data
- If data doesn't support user theme, discuss gap transparently
- Remember: User knows business, agent structures knowledge

### If Inconsistencies Found

If Finance Analyst or Information Vault flag inconsistencies during storyline development:
1. Immediately surface to user
2. Don't include contradictory claims in storyline
3. Investigate which source is correct
4. If unresolved, acknowledge in risk section or exclude entirely

## Best Practices

1. **Listen More Than Tell**: User input is critical. Facilitate, don't dictate.

2. **Show Your Work**: Always show supporting evidence for proposed themes.

3. **Iterate**: Expect 2-3 rounds of refinement on themes and thesis.

4. **Validate Early**: Test buyer relevance in Act 2, not after full construction.

5. **Evidence First**: Never construct narrative before gathering data from agents.

6. **Save Frequently**: Save storyline brief after each act in case of interruption.

7. **Be Honest About Gaps**: Better to acknowledge missing data than fake it.

8. **Stay Focused**: If workshop gets derailed, bring back to core question: "What's the investment story?"

## Workflow Customization by Depth Level

### Quick (30 min)
- Act 1: Abbreviated (10 min)
- Act 2: Core themes only, minimal refinement (10 min)
- Act 3: Thesis statement only, skip full arc (5 min)
- Act 4: Quick coherence check only (3 min)
- Act 5: Storyline brief + thesis only (2 min)

### Standard (60 min)
- Full 5-act process as described above
- All outputs except CIM narrative outline

### Comprehensive (90+ min)
- Full 5-act process with deep exploration
- All outputs including CIM narrative outline
- Additional buyer persona testing
- Multiple rounds of refinement
